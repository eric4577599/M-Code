## 4. 收尾:停掉舊 m-code connector

順序:**先確認 Mac mini 對外 200,再停舊的**(避免中間完全斷線)。
原因:同一 tunnel 兩個 connector 同時活著 → edge 負載平衡輪到壞的那個 → 間歇性 502。

### 判斷舊筆電形態(在舊 Windows 上)
```powershell
Get-Process cloudflared -ErrorAction SilentlyContinue                 # A. 原生 cloudflared(VBS)
docker ps --filter "name=cloudflared-mcode" --filter "name=m-code-web" # B. Docker 容器
```

### 形態 A:Windows 啟動腳本(VBS + 原生 cloudflared)
1. 擋開機自啟:`explorer "shell:startup"` → 找 `m-detector-startup.vbs`,剪下備份(勿直接刪)。
   - VBS 內有**兩個** tunnel,第二個才是 m-code(`2cad6e8b-…`)。若 m-detector 仍要留在這台,**只刪 m-code 那一行**,勿整支移除。
2. 停目前在跑的 m-code cloudflared:
   ```powershell
   Get-CimInstance Win32_Process -Filter "Name='cloudflared.exe'" | Select-Object ProcessId, CommandLine
   Stop-Process -Id <PID> -Force   # 只停 m-code 那支,勿誤停 m-detector
   ```

### 形態 B:Docker 容器
```powershell
docker stop cloudflared-mcode m-code-web
docker rm   cloudflared-mcode m-code-web
docker update --restart=no cloudflared-mcode m-code-web 2>$null
# 若用 compose:到對應目錄 docker compose down
```

### 收尾驗證
1. Cloudflare Zero Trust → Tunnels → `m-code`:應只剩 **1 個 connector(Mac mini)HEALTHY**,舊筆電 connector 掉成 inactive。
2. 連續打 10 次對外都 200 才算乾淨切換:
   ```
   for /l %i in (1,1,10) do @curl -s -o nul -w "%%{http_code}\n" https://m-code.ericchh.work/demo/index.html
   ```

> ⚠️ 陷阱:m-code 與 m-detector 若共用同一 VBS / 同一 Docker 環境,**只停 m-code 部分**,勿一併關掉 m-detector。

## 6. 實際部署結果(2026-07-01 完成上線)

### 遭遇:外接碟 FDA 雷(印證 CLAUDE.md §4.6)
原訂用 `setup-macmini.sh` 的**系統 LaunchDaemon(需 sudo)**,但遇到兩個阻礙:
1. `sudo` 在非互動環境讀不到密碼(`a terminal is required to read the password`)。
2. 改用 launchd 起 web server 時,從外接碟 serve 直接 **`PermissionError: Operation not permitted`** —— launchd 起的 python 程序**沒有 FDA,讀不到 `/Volumes/G70Pro/`**。tunnel 不碰外接碟故不受影響。

### 採用方案:使用者層 LaunchAgent(免 sudo)+ 內接碟 docroot
| 元件 | 作法 |
|---|---|
| 靜態檔 docroot | `rsync` 把 `demo/` + `dist/` 等複製到**內接碟** `/Users/hezhaoxing/m-code-site`(避開外接碟 FDA),repo 仍留外接碟開發 |
| web server | LaunchAgent `~/Library/LaunchAgents/com.ericchh.mcode-web.plist` → `python3 -m http.server 8765 --bind ::` serve 內接碟 docroot |
| tunnel | LaunchAgent `~/Library/LaunchAgents/com.ericchh.mcode-tunnel.plist` → `cloudflared tunnel run --token <token>`(token 寫入 plist,權限 600) |
| 載入 | `launchctl bootstrap gui/$(id -u) <plist>`(免 sudo) |

### 驗收(全 200)
- 本機 `localhost:8765` /demo/index.html、/demo/mobile.html、/dist/index.js → **200**
- 對外 `https://m-code.ericchh.work` 同三路徑 → **200**(1033 已消失)
- cloudflared 4 連線(khh01/tpe01);KeepAlive 韌性測試:殺 web 程序後自動重啟、仍 200 ✅

### 與官方 sudo 版的關係 / 維運
- 本方案為 **使用者層**:使用者登入後自動起、斷線/崩潰自動重連;但**開機未登入前不會跑**(系統 LaunchDaemon 才有)。Mac mini 若自動登入即等效。
- **改完程式碼生效**:在外接碟 repo `npm run build` 後,需 `rsync -a --delete --exclude node_modules --exclude .git "/Volumes/G70Pro/cusor pool/M-Code/" /Users/hezhaoxing/m-code-site/`(因 docroot 是內接碟副本)。
- **若日後要轉成官方系統 LaunchDaemon(`setup-macmini.sh`)**:先卸載本 agent 避免搶 8765 / 搶 tunnel:
  ```bash
  launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.ericchh.mcode-web.plist
  launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.ericchh.mcode-tunnel.plist
  ```
  且該 daemon 同樣有外接碟 FDA 問題 → 建議 docroot 也指內接碟,或手動把 python3 加進 FDA。

### 拆除(完全移除本次部署)
```bash
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.ericchh.mcode-web.plist
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.ericchh.mcode-tunnel.plist
rm ~/Library/LaunchAgents/com.ericchh.mcode-{web,tunnel}.plist
rm -rf ~/m-code-site
```
